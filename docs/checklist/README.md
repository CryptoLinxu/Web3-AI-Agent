# Project Checklist 目录

本目录维护项目的完整功能清单和后续规划，用于项目自我进化和持续跟踪。

## 文件说明

### PROJECT-CHECKLIST.md

项目主清单文档，包含：

1. **已完成功能** - 所有已实现的功能模块
2. **进行中功能** - 正在开发中的功能
3. **未完成功能** - MVP 范围内待实现的功能（按优先级 P0/P1/P2 排序）
4. **未来规划** - MVP 范围外的增强功能（短期/中期/长期）
5. **技术债务** - 需要重构和优化的模块
6. **项目演进路线** - 可视化的发展路径
7. **关键指标** - 项目健康度量化指标
8. **下一步行动建议** - 具体可执行的优先级建议

## 自动触发条件

Project Checklist 会在以下场景自动更新：

### 关键词触发
- 更新 checklist
- 项目现状
- 项目进度
- 后续规划
- 未来计划
- 已完成哪些
- 未完成哪些
- 下一步做什么

### 命令触发
- `/project-checklist`
- `/checklist`

### 上下文触发
- 交付型任务完成后（FEAT/PATCH/REFACTOR）
- 用户询问"目前项目进展如何"
- 用户询问"接下来应该做什么"

- 下一步建议（P0/P1/P2）

### 职责划分

- **Changelog**：记录"改了什么"（变更事实）
- **Checklist**：记录"完成了什么"和"要做什么"（功能清单）
- **Digest**：记录"学到了什么"（经验教训）
- **Update-Map**：记录"项目状态"（技能地图）

## 维护原则

1. **基于事实**：必须基于实际代码状态，不虚构完成情况
2. **定期更新**：交付型任务完成后自动更新
3. **优先级明确**：P0/P1/P2 分级清晰
4. **可执行性**：下一步建议必须具体可执行
5. **技术债务透明**：明确记录待重构和优化项
6. **历史可追溯**：保留更新历史记录
