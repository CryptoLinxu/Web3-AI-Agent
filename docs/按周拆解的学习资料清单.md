# 按周拆解的学习资料清单

## 目标

本清单服务于 `Web3 AI Agent` 项目的 30 天转型与落地过程，目标不是系统性学完 AI，而是围绕项目开发节奏建立最小可用知识地图。

使用原则：

- 每周只学习当前阶段必需内容，避免知识过载。
- 学习必须服务于项目产出，不能脱离代码和文档单独进行。
- 每周结束时，应形成可复用的阶段结论，并交给后续 skill 继续推进。

---

## 第 1 周：LLM API、Chat 与 Function Calling

### 本周目标

建立 `LLM + Messages + Tool Calling` 的最小认知闭环，并能在项目中完成一次真实的工具调用。

### 需要学习的内容

- Chat Completions / Responses API 的基础调用方式
- `system`、`user`、`assistant` 消息角色
- 流式输出的基本原理与前端处理方式
- Function Calling / Tool Calling 的结构化定义方式
- JSON Schema 在工具参数描述中的作用

### 推荐学习主题

- 什么是 Agent 最小闭环
- 为什么模型不是在执行代码，而是在生成调用指令
- 消息上下文如何影响后续回答
- 如何定义一个“可被模型可靠调用”的工具

### 本周项目产出

- 一个最小聊天页面
- 一个支持流式输出的基础对话接口
- 一个 `getETHPrice` 工具
- 一个 `getWalletBalance` 工具
- 一篇项目内学习笔记，说明 Tool Calling 的工作原理

### 本周验收标准

- 能解释 `tool calling` 的工作机制
- 能说明普通 Chat 与 Agent 的差异
- 项目内已出现至少 2 个真实可调用工具
- 可以用自然语言触发一次完整的工具调用

### 后续关联的 skill 使用入口

- `skills/x-ray/origin`
- `skills/x-ray/check-in`
- `skills/x-ray/prd`

## 第 2 周：Agent Loop、Prompt 设计与 Web3 工具接入

### 本周目标

从“模型能调用工具”升级到“系统能完成一轮任务闭环”，并开始接入真实 Web3 数据能力。

### 需要学习的内容

- Agent Loop 的基本结构
- 工具结果回填给模型的方式
- Prompt 的角色设定、边界约束与输出格式控制
- `ethers` 的基础调用
- RPC、余额查询、Gas、Token 信息查询

### 推荐学习主题

- ReAct 风格的“思考-行动-观察”
- 何时应该直接回答，何时应该调工具
- Web3 数据查询与通用 API 查询的差异
- 工具失败和参数错误的降级处理

### 本周项目产出

- Agent Loop v1
- 至少 3 个 Web3 工具
- 一份工具契约清单
- 一张系统架构草图

### 本周验收标准

- 能解释为什么 Agent 需要 Loop
- 工具调用结果会被模型重新消费并生成最终回复
- 至少支持查询价格、查询钱包余额、查询 Gas/Token 信息中的任意三项
- 工具异常时系统不会直接崩溃

### 后续关联的 skill 使用入口

- `skills/x-ray/req`
- `skills/x-ray/architect`
- `skills/x-ray/qa`

## 第 3 周：Memory、RAG 入门与产品边界

### 本周目标

让项目从 Demo 变成初步产品，具备记忆、知识增强和边界控制意识。

### 需要学习的内容

- 短期 Memory 与长期 Memory 的差异
- Conversation History 的裁剪与管理
- RAG 的基础流程：切片、检索、拼接、生成
- AI 产品中的风险边界与安全护栏

### 推荐学习主题

- Memory 与上下文消息的关系
- 为什么 RAG 比“把所有信息塞进 Prompt”更可靠
- Web3 场景下的高风险误导类型
- 如何定义“不回答”“不建议”“只给数据不做判断”的策略

### 本周项目产出

- 简单用户偏好记忆方案
- 一个知识增强的预留设计
- 风险提示与免责声明策略
- 一版稳定的系统 Prompt

### 本周验收标准

- 能区分短期上下文和长期记忆
- 能讲清楚 RAG 的最小流程
- 输出中带有必要的风险提示
- 明确记录了当前系统“不做什么”

### 后续关联的 skill 使用入口

- `skills/x-ray/check-in`
- `skills/x-ray/architect`
- `skills/x-ray/audit`

### 本周目标

- README、项目架构图、简历项目描述写法

### 推荐学习主题

- 如何用一页 README 讲清项目价值
- 如何把项目包装成简历可讲案例

### 本周项目产出

- 在线可访问版本
- 完整 README
- 项目架构图
- 简历项目描述
- 一篇项目复盘文章

### 本周验收标准

- 有可访问演示地址或清晰本地运行文档
- README 可帮助陌生人理解项目和启动项目
- 能输出 1 分钟版和 3 分钟版项目介绍
- 已形成下一阶段继续迭代的路线图

### 后续关联的 skill 使用入口

- `skills/x-ray/qa`

## 使用方法

建议每周采用以下节奏：

1. 周初用 `check-in` 明确本周要解决的问题与知识点。
2. 周中按 `architect / qa / coder` 节奏推进技术设计与实现。
3. 周末使用 `digest` 做复盘，沉淀问题、知识和下一步行动。

本清单不等于一次性阅读材料库，而是项目推进过程中的学习导航文档。
