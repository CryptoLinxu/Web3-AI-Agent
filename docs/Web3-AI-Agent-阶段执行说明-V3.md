# Web3 AI Agent 阶段执行说明 V3

## 使用方式

V3 中，“阶段”仍然存在，但“阶段”不等于“每次都完整跑一遍”。

正确理解是：

1. `origin` 先判断任务类型
2. 只有交付型任务才进入 `pipeline`
3. 只有实施型任务才强制进入 `check-in`
4. 不同任务只引用自己需要的阶段

## V3 的阶段和任务关系

- `DISCOVER` 主要使用探索阶段，不进入实施阶段
- `BOOTSTRAP` 主要使用初始化和地图建立阶段
- `DEFINE` 主要使用目标澄清和边界定义阶段
- `FEAT` 最完整，会覆盖定义、设计、实现、审计、沉淀
- `PATCH` 主要覆盖修复、验证、沉淀
- `REFACTOR` 重点覆盖结构设计、回归验证、审计

## V3 的统一执行规则

1. 不是所有任务都走全阶段。
2. 只有实施型任务必须经过 `check-in`。
3. `PATCH` 默认走快链路。
4. `REFACTOR` 默认优先保障结构和回归。
5. `FEAT` 默认优先保障边界和验收。
6. 如果发现范围变化，优先回退 `prd / req`。
