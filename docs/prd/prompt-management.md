# 提示词管理功能 PRD

## 背景

当前项目的 System Prompt 硬编码在 `apps/web/app/api/chat/route.ts` 中（第 160-206 行），存在以下问题：
1. 提示词分散在代码中，不便于修改和维护
2. 无法快速预览和使用不同场景的提示词
3. 用户无法快速选择预设的提示词模板
4. 输入框下方的快捷键提示布局不够合理

## 目标

1. **集中管理提示词**：将所有场景的提示词提取到独立的配置文件中，支持随时修改
2. **优化输入框布局**：将快捷键提示移到右侧靠近发送按钮
3. **提供提示词选择器**：在输入框下方左侧增加入口，弹窗展示按类分组的提示词列表，支持一键填充到输入框

## 用户场景

### 场景 1：开发者修改提示词
- **用户**：开发者
- **需求**：快速找到并修改某个场景的提示词
- **现状**：需要搜索代码文件，找到硬编码的字符串
- **期望**：在配置文件中按分类查找，直接修改

### 场景 2：用户快速使用预设提示词
- **用户**：终端用户
- **需求**：快速使用常见的查询模板（如“查询 ETH 价格”、“查询我的余额”）
- **现状**：需要手动输入完整内容
- **期望**：点击按钮，提示词自动填充到输入框，**用户可修改后发送或直接发送**

### 场景 3：查看可用提示词
- **用户**：终端用户
- **需求**：了解系统支持哪些查询场景
- **现状**：只能通过欢迎消息或试错了解
- **期望**：通过弹窗直观查看所有分类和提示词

## 范围

### 包含功能

#### 1. 提示词配置系统
- 创建独立的提示词配置文件（如 `apps/web/config/prompts.ts`）
- 按场景分类组织提示词：
  - **价格查询**：ETH 价格、BTC 价格等
  - **余额查询**：查询我的余额、查询指定地址余额等
  - **Gas 查询**：查询以太坊 Gas 等
  - **Token 查询**：查询 Token 信息、查询 Token 余额等
  - **转账操作**：转账 ETH、转账 USDT 等
  - **系统提示词**：SYSTEM_PROMPT_BASE（AI 行为定义）
- 每个提示词包含：
  - `id`: 唯一标识
  - `category`: 分类
  - `title`: 标题（展示用）
  - `content`: 提示词内容
  - `description`: 描述（可选）

#### 2. 输入框布局优化
- 将"Enter 发送 · Shift+Enter 换行"从左侧移到右侧
- 靠近发送按钮，与"数据仅供参考"提示并列
- 保持字体大小和样式不变

#### 3. 提示词选择器组件
- **入口按钮**：在输入框下方左侧添加图标按钮（如模板/快捷方式图标）
- **弹窗设计**：
  - 丝滑动画（淡入 + 缩放）
  - 按分类分组展示提示词
  - 每个提示词项包含：标题、描述（可选）、使用按钮
  - 使用按钮：圆形按钮，带 → 箭头 icon，悬停有高亮效果
- **交互逻辑**：
  - 点击使用按钮：将提示词内容填充到输入框，关闭弹窗
  - **填充后用户可自由修改，不自动发送**
  - 点击遮罩层或 ESC 键：关闭弹窗
  - 支持滚动查看所有分类

### 非目标

1. **不修改 AI 的 System Prompt 逻辑**：
   - 仅管理用户输入的提示词模板
   - SYSTEM_PROMPT_BASE 仍由后端使用，但提取到配置文件
   
2. **不支持用户自定义提示词**：
   - 仅提供预设模板
   - 不涉及用户创建、编辑、保存自定义提示词

3. **不修改后端 API 结构**：
   - 提示词配置仅在前端使用
   - 不改变 chat API 的请求格式

4. **不实现提示词变量替换**：
   - 提示词为静态文本
   - 不涉及动态变量（如 {address}、{amount}）的替换逻辑

## 风险边界

### 技术风险
1. **提示词配置文件体积**：如果提示词过多，可能影响首屏加载
   - **缓解**：使用懒加载或按需导入

2. **弹窗性能**：如果提示词分类和数量很多，可能影响渲染性能
   - **缓解**：使用虚拟列表或分页（初期预计 < 50 个提示词，无需优化）

### 用户体验风险
1. **误触填充覆盖输入内容**：用户可能误点提示词按钮，导致输入框内容被覆盖
   - **缓解**：填充后自动聚焦，用户可立即修改或撤销（无需确认框，保持流畅体验）

2. **弹窗遮挡**：在小屏幕设备上，弹窗可能遮挡输入框
   - **缓解**：响应式设计，移动端使用底部抽屉而非居中弹窗

## 技术约束

1. **配置文件格式**：使用 TypeScript 对象导出，便于类型检查
2. **弹窗实现**：使用 React Portal 或固定定位，避免 z-index 冲突
3. **动画实现**：使用 CSS transition 或 Tailwind 动画类
4. **状态管理**：使用 React useState 管理弹窗开关状态
5. **图标使用**：使用内联 SVG，不引入新图标库
