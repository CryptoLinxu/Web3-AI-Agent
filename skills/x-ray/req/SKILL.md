# Req

## 适用场景

- 把 PRD 拆成执行项
- PATCH 的默认入口
- REFACTOR 的默认入口

## 输入

- PRD
- bug 描述
- 重构目标

## 输出

按类型输出：

- `FEAT` 需求卡
- `PATCH` 缺陷卡
- `REFACTOR` 重构卡

统一包含：
- 来源
- 目标
- 影响范围
- 依赖关系
- 验收标准
- 下一跳

## 流程

1. 确定任务对象
2. 拆成最小交付单元
3. 写清影响范围
4. 写清验收条件

## 边界

- 不产出架构说明
- 不写代码

## 衔接

- 进入 `check-in`

## 规则

1. `PATCH` 默认从 `req` 开始。
2. `REFACTOR` 默认从 `req` 开始。
3. 如果任务卡仍然过大，应继续拆分。
