# Web3 AI Agent Skill Templates V3

## 1. `check-in` 总模板

所有实施型任务在进入设计或编码前，都必须先输出下面这份模板。

```markdown
# {任务类型}-{编号或主题} Check-In

## 本阶段要解决的问题

## 本阶段必须掌握的上下文

## 本阶段采用的方案

## 本阶段不做什么

## 本阶段产物

## 本阶段完成标准

## 进入下一阶段前要调用的 skill
```

## 2. FEAT 场景 `check-in`

```markdown
# FEAT-{编号} Check-In

## 本阶段要解决的问题
- 本次新增什么能力
- 这个能力服务谁
- 主路径是什么

## 本阶段必须掌握的上下文
- 业务规则
- 数据来源
- 依赖模块
- 异常场景

## 本阶段采用的方案
- 主路径方案
- 数据方案
- 降级方案

## 本阶段不做什么
- 非目标
- 暂不覆盖的扩展能力

## 本阶段产物
- 需求卡
- 架构说明
- 测试清单
- 代码实现

## 本阶段完成标准
- 主路径可跑通
- 数据来源清楚
- 失败场景可解释
- 验收标准可验证

## 进入下一阶段前要调用的 skill
- architect
- qa
- coder
```

## 3. PATCH 场景 `check-in`

```markdown
# PATCH-{编号} Check-In

## 本阶段要解决的问题
- 具体 bug 是什么
- 复现条件是什么
- 预期行为是什么
- 根因假设是什么

## 本阶段必须掌握的上下文
- 出错模块
- 相关状态流或数据流
- 回归风险点

## 本阶段采用的方案
- 最小修复路径
- 是否补防御逻辑
- 是否补测试

## 本阶段不做什么
- 不顺手重构
- 不扩大修复范围

## 本阶段产物
- 缺陷卡
- 修复代码
- 回归验证记录

## 本阶段完成标准
- bug 稳定修复
- 相关回归项通过
- 无新增行为偏差

## 进入下一阶段前要调用的 skill
- coder
- qa
- audit（高风险时）
```

## 4. REFACTOR 场景 `check-in`

```markdown
# REFACTOR-{编号} Check-In

## 本阶段要解决的问题
- 当前结构问题是什么
- 为什么现在要重构
- 哪些行为必须保持不变

## 本阶段必须掌握的上下文
- 模块边界
- 依赖关系
- 兼容约束
- 性能或维护性瓶颈

## 本阶段采用的方案
- 新结构方案
- 迁移路径
- 兼容方案

## 本阶段不做什么
- 不改变的对外行为
- 不顺手新增功能

## 本阶段产物
- 重构卡
- 架构说明
- 回归测试清单
- 重构代码

## 本阶段完成标准
- 行为等价
- 结构更清晰
- 回归项通过
- 风险已记录

## 进入下一阶段前要调用的 skill
- architect
- qa
- coder
- audit
```
