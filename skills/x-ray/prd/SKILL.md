# PRD

## 适用场景

- FEAT 的正式边界定义
- 重构影响产品边界时
- bug 根因其实是需求错误时

## 输入

- `pm` 输出或明确目标
- 用户场景
- 约束与非目标

## 输出

```markdown
# {主题} PRD

## 背景
## 目标
## 用户场景
## 范围
## 非目标
## 风险边界
## 验收标准
```

## 流程

1. 明确做什么
2. 明确不做什么
3. 明确验收标准
4. 明确风险边界

## 边界

- 不做技术方案
- 不直接拆成代码任务

## 衔接

- 进入 `req`

## 规则

1. `PRD` 的重点是边界，不是实现。
2. 没有清晰非目标的 PRD 视为未完成。
