# Digest

## 作用

把一轮任务中真正值得保留的经验沉淀下来，而不是只记录“改了哪些文件”。

## 输入

- 本轮产物
- QA 结果
- Audit 结论

## 输出

```markdown
# {任务} 复盘

## 本轮完成了什么
## 遇到了什么问题
## 学到了什么
## 仍未解决的问题
## 下一步建议
```

## 流程

1. 总结完成项
2. 总结偏差和问题
3. 记录经验和限制
4. 给出下一步建议

## 边界

- 不代替地图更新
- 不替代需求文档

## 衔接

- 进入 `update-map`

## 规则

1. 重点记录“为什么卡住/为什么成功”，而不是流水账。
2. `PATCH` 可以轻量写，但不建议省略。
