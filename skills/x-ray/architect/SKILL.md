# Architect

## 适用场景

- 有接口变化
- 有状态流变化
- 有模块边界变化
- 有结构性重构

## 输入

- `check-in`
- 任务卡

## 输出

```markdown
# {主题} 架构说明

## 目标
## 模块边界
## 数据流
## 消息流
## 接口契约
## 错误处理
## 风险点
```

## 流程

1. 确定影响模块
2. 定义边界与契约
3. 补主路径和异常路径

## 边界

- 不直接写测试
- 不直接承担编码

## 衔接

- 进入 `qa` 或 `coder`

## 规则

1. 如果只是纯局部修补且无结构变化，可跳过。
2. 如果发现需求边界变化，回退 `prd / req`。
