# Update-Map

## 作用

维护当前项目状态，让下一轮任务能基于最新上下文继续推进。

## 输入

- 当前任务结果
- 新增或修改的文档
- 新增或修改的能力

## 输出

```markdown
## Map 更新
- 当前状态：
- 影响模块/能力：
- 新增文档：
- 需要关注的后续入口：
```

## 流程

1. 更新任务状态
2. 更新索引或地图
3. 更新下一步建议

## 边界

- 不做深度复盘
- 不重新定义需求

## 衔接

- 返回 `origin`

## 规则

1. 任何改变状态的交付任务都应更新地图。
2. `digest` 负责经验，`update-map` 负责状态，不混写。
