## 模块职责

### 1. 用户层 (apps/web)

| 模块 | 职责 | 关键文件 |
|------|------|----------|
| Chat UI | 聊天界面、用户输入、消息展示 | `app/page.tsx` |
| Message List | 消息列表渲染、流式输出展示 | `components/MessageList.tsx` |
| Tool Result | 工具调用结果可视化 | `components/ToolResult.tsx` |

### 2. API 层

| 路由 | 职责 | 关键文件 |
|------|------|----------|
| `/api/chat` | 处理对话请求，协调 Agent Loop | `app/api/chat/route.ts` |
| `/api/tools` | 工具调用接口 | `app/api/tools/route.ts` |
| `/api/health` | 健康检查 | `app/api/health/route.ts` |

### 3. Agent Core 层

| 模块 | 职责 | 说明 |
|------|------|------|
| Intent Classifier | 识别用户意图，判断是否需要工具 | 基于 LLM 的意图分类 |
| Agent Loop | 主循环：理解 → 决策 → 执行 → 回复 | ReAct 模式简化版 |
| Memory Manager | 会话上下文管理 | 最小实现，仅保留当前会话 |

### 4. 工具层 (packages/web3-tools)

| 工具 | 功能 | 输入 | 输出 |
|------|------|------|------|
| `getETHPrice` | 查询 ETH 实时价格 | 无 | 价格、数据源、时间戳 |
| `getWalletBalance` | 查询地址 ETH 余额 | address | 余额、单位、数据源 |
| `getGasPrice` | 查询当前 Gas 价格 | 无 | gasPrice、建议值 |

### 5. 数据层

| 数据源 | 用途 | 配置项 |
|--------|------|--------|
| OpenAI API | LLM 对话、意图识别 | `OPENAI_API_KEY` |
| Alchemy/Infura | 链上数据查询 | `ALCHEMY_API_KEY` |
| Price API | 价格数据 | 第三方价格接口 |

## 数据流

### 正常对话流程

```
用户输入 → Intent Classifier → (无需工具) → LLM 直接回复 → 展示给用户
```

### 工具调用流程

```
用户输入 → Intent Classifier → 需要工具
                              ↓
                    Agent Loop 决策调用哪个工具
                              ↓
                    调用 Web3 Tool → 获取结果
                              ↓
                    结果回填 LLM → 生成回复
                              ↓
                    展示给用户 + 保留上下文
```

## 错误处理策略

| 错误类型 | 处理方式 | 用户感知 |
|----------|----------|----------|
| 工具参数无效 | 返回错误提示，请求澄清 | "请提供有效的钱包地址" |
| 工具执行失败 | 降级为说明性回复 | "查询失败，建议稍后重试" |
| API 超时 | 超时提示，建议重试 | "请求超时，请重试" |
| 超出能力边界 | 明确告知限制 | "此问题超出我的能力范围" |

## 安全边界

1. **不执行写操作**：所有工具均为只读查询
2. **不给出交易建议**：高风险问题返回数据参考 + 风险提示
3. **数据来源透明**：明确标注数据来自工具查询
4. **免责声明**：所有回复附带风险提示

## 技术选型理由

| 技术 | 选型理由 |
|------|----------|
| Next.js 14 | 全栈框架，API Routes 支持，Vercel 部署友好 |
| TypeScript | 类型安全，更好的 IDE 支持 |
| Tailwind CSS | 原子化 CSS，快速 UI 开发 |
| ethers.js | Web3 标准库，文档完善 |
| OpenAI API | 成熟的 LLM API，Tool Calling 支持好 |

## 扩展预留

- **多链支持**：工具层预留 chainId 参数
- **RAG 接入**：预留知识库接口位置
- **持久化 Memory**：预留数据库接口
- **更多工具**：工具层可插拔设计
