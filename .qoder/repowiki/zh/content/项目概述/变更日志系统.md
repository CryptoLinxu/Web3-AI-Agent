## 简介

变更日志系统是 Web3 AI Agent 项目中用于记录和追踪代码变更历史的重要基础设施。该系统采用标准化的文档格式，为 AI 和开发者提供完整的变更上下文，支持自动化的变更记录生成和手动补录功能。

系统的核心目标是：
- 提供完整的项目演进历史记录
- 支持 AI 上下文理解和开发者追溯
- 实现自动化和手动相结合的变更记录机制
- 建立标准化的变更分类和文档规范

**更新** 新增了对多链Web3工具重构的详细记录，包括四阶段架构重构的完整实现和最终能力矩阵。

### 多链Web3工具重构组件

**新增** 多链Web3工具重构是本次更新的核心功能，实现了四阶段的架构重构，支持5条链的统一查询能力。

#### 四阶段架构重构

##### Phase 1: 价格工具抽象（REFACTOR）
- **新增** `TokenPriceData` 类型统一价格数据
- **新增** `SYMBOL_MAP` 币种映射（ETH, BTC, SOL, MATIC, BNB）
- **新增** `getTokenPrice(symbol)` 参数化函数
- **修改** 合并 getETHPrice/getBTCPrice 为 getTokenPrice

##### Phase 2: EVM 多链支持（FEAT）
- **新增** `chains/config.ts` 链配置管理
- **新增** `chains/evm-adapter.ts` EVM 适配器
- **新增** `EvmChainId` 类型（ethereum, polygon, bsc）
- **修改** `getBalance` 支持链参数化

##### Phase 3: BTC/Solana 支持（FEAT）
- **新增** `chains/bitcoin.ts` BTC 适配器，使用 Blockchain.info/Blockchair API
- **新增** `chains/solana.ts` Solana 适配器，使用 JSON-RPC API
- **新增** `NonEvmChainId` 类型（bitcoin, solana）
- **修改** `ChainId` 统一类型

##### Phase 4: Token 信息查询（FEAT）
- **新增** `tokens/registry.ts` Token 注册表，包含11个主流Token
- **新增** `token.ts` Token 查询工具
- **新增** `TokenMetadata` 接口

#### 链适配器模式

多链架构采用适配器模式设计，定义了统一的 ChainAdapter 接口：

```typescript
interface ChainAdapter {
  getBalance(address: string): Promise<ToolResult<BalanceData>>
  getGasPrice?(): Promise<ToolResult<GasData>>  // 仅 EVM
  validateAddress(address: string): boolean
}

// EVM 适配器
class EvmChainAdapter implements ChainAdapter

// BTC 适配器
class BitcoinAdapter implements ChainAdapter

// Solana 适配器
class SolanaAdapter implements ChainAdapter
```

#### 统一类型系统

重构后的类型系统提供了完整的链类型支持：

```typescript
// EVM 兼容链类型
type EvmChainId = 'ethereum' | 'polygon' | 'bsc'

// 非 EVM 链类型
type NonEvmChainId = 'bitcoin' | 'solana'

// 统一链 ID 类型
type ChainId = EvmChainId | NonEvmChainId

// 链配置（EVM）
interface EvmChainConfig {
  id: EvmChainId
  name: string
  nativeToken: string
  chainId: number
  rpcUrls: string[]
  explorerUrl: string
}

// 链配置（非 EVM）
interface NonEvmChainConfig {
  id: NonEvmChainId
  name: string
  nativeToken: string
  apiUrls: string[]
  explorerUrl: string
}
```

## 故障排除指南

### 常见问题及解决方案

#### 变更记录未生成
**症状**：执行 pipeline 后未生成变更记录
**可能原因**：
- 环境变量配置错误
- 权限不足
- 网络连接问题

**解决方案**：
1. 检查环境变量配置
2. 验证权限设置
3. 确认网络连接状态

#### 索引不准确
**症状**：索引文件与实际文档不匹配
**可能原因**：
- 手动修改了文件名
- 未更新索引文件
- 文件编码问题

**解决方案**：
1. 按照命名规范重命名文件
2. 手动更新索引文件
3. 检查文件编码格式

#### 文档格式错误
**症状**：文档无法正确渲染
**可能原因**：
- Markdown 语法错误
- 缺少必需字段
- 格式不规范

**解决方案**：
1. 使用 Markdown 校验工具
2. 检查必需字段完整性
3. 参考标准模板格式

**更新** 新增了多链架构相关的故障排除指南：

#### 多链查询问题
**症状**：多链查询功能异常或返回错误
**可能原因**：
- 链配置错误
- RPC 节点不可用
- 地址格式不正确
- Token 注册表缺失

**解决方案**：
1. **链配置错误**：检查 `ChainConfig` 配置和 `SYMBOL_MAP` 映射
2. **RPC 节点不可用**：验证 RPC URL 可访问性和网络连接
3. **地址格式不正确**：使用适配器的 `validateAddress` 方法验证地址
4. **Token 注册表缺失**：确认 Token 符号或合约地址在注册表中

#### 类型安全问题
**症状**：TypeScript 编译错误或运行时类型错误
**可能原因**：
- ChainId 类型不匹配
- 未正确使用类型守卫函数
- 泛型参数错误

**解决方案**：
1. **ChainId 类型不匹配**：使用 `isEvmChain` 类型守卫进行类型窄化
2. **类型守卫函数**：确保在分支判断中正确使用类型守卫
3. **泛型参数**：检查泛型约束和类型推断

#### 向后兼容问题
**症状**：旧 API 调用失败或行为异常
**可能原因**：
- @deprecated 函数调用
- 参数变化
- 返回值格式变化

**解决方案**：
1. **@deprecated 函数**：迁移到新的参数化函数
2. **参数变化**：检查新函数的参数要求
3. **返回值格式**：使用新的 `ToolResult` 格式

#### 流式输出问题
**症状**：SSE 流式输出功能异常
**可能原因**：
- 浏览器兼容性问题
- 网络连接中断
- 服务器配置错误

**解决方案**：
1. 检查浏览器对 EventSource 的支持
2. 验证网络连接稳定性
3. 确认服务器 SSE 配置正确
